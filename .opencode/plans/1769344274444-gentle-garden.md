# Refactor raw_agent_handler to use StreamChunk struct

## Overview
Refactor `raw_agent_handler` in `src/main.rs` to use the `StreamChunk` struct from `src/streaming.rs` instead of raw byte manipulation. Add helper methods to `StreamChunk` and restructure the handler with early returns to reduce nesting.

## Changes Required

### 1. Add Helper Methods to `StreamChunk` (src/streaming.rs:14-80)

Add the following methods to the `impl StreamChunk` block:

```rust
// Parses a byte chunk and returns the data portion
// Returns an error if the chunk is invalid
pub fn parse_data(bytes: &[u8]) -> Result<Vec<u8>, String>

// Similar to parse_data but returns the full StreamChunk for access to flags
pub fn try_from_bytes_with_data(bytes: &[u8]) -> Result<(Self, Vec<u8>), String>
```

**Rationale**: The handler currently needs to:
- Check if a chunk is valid (magic bytes, length)
- Extract the `is_error` flag
- Extract the data portion

Adding these methods encapsulates the parsing logic and makes the handler cleaner.

### 2. Refactor `raw_agent_handler` (src/main.rs:330-410)

**Current Issues**:
- Manual byte manipulation at lines 357-364 (checking length and extracting `is_error` flag)
- Manual byte slicing at lines 373-377 and 387-390 (extracting data)
- Nested control flow: match -> match -> if-else -> if

**Refactoring Approach**:

#### a. Extract validation and parsing into helper functions
```rust
// Parse first chunk and return StreamChunk or error
fn parse_first_chunk(chunk_bytes: Vec<u8>) -> Result<StreamChunk, &'static str>

// Create error response helper
fn create_error_response(status: StatusCode, error: String) -> Response
```

#### b. Flatten with early returns
```rust
async fn raw_agent_handler(...) -> impl IntoResponse {
    let (response_sender, mut response_receiver) = tokio::sync::mpsc::channel::<Vec<u8>>(32);

    // Early return on command send failure
    if call_t!(...).is_err() {
        return error_response(...);
    }

    // Early return on no first chunk
    let first_chunk_bytes = match response_receiver.recv().await {
        Some(chunk) => chunk,
        None => return error_response(...),
    };

    // Early return on parse failure
    let first_chunk = match StreamChunk::from_bytes(&first_chunk_bytes) {
        Ok(chunk) => chunk,
        Err(_) => return error_response(...),
    };

    // Early return on error flag
    if first_chunk.is_error {
        return error_response(...);
    }

    // Stream the rest...
}
```

**Benefits**:
- Eliminates 3-4 levels of nesting
- Uses type-safe `StreamChunk` parsing with proper validation
- Reduces code duplication in data extraction
- Easier to read and maintain
