# Fix Raw Download API to Stream with Backpressure

## Problem
The raw download REST API buffers entire files in memory before sending the HTTP response. This causes memory issues for large files and provides no backpressure when the HTTP client is slow.

**Current Issues:**
- Collects ALL chunks into `all_data` Vec before sending (line 347, 359 in main.rs)
- Uses `unbounded_channel` with no flow control (line 333 in main.rs, line 16, 90 in router.rs)
- Sends entire file at once: `Body::from(all_data)` (line 375 in main.rs)

## Solution
Replace buffering with streaming response using Axum's `Body::from_stream()` and bounded channels for backpressure.

## Files to Modify

1. **src/main.rs** (lines 329-388) - `raw_agent_handler` function
2. **src/actors/router.rs** (line 16, 90, 345) - Router state and message types
3. **src/bin/redoor-agent.rs** (line 307, 314-320, 117) - Agent WebSocket channel and send handling

### Step 2: Rewrite raw_agent_handler (src/main.rs)

Replace the entire handler (lines 329-388) with streaming implementation:

```rust
async fn raw_agent_handler(
    Path((agent, path)): Path<(String, String)>,
    AxumState(state): AxumState<ServerState>,
) -> impl IntoResponse {
    // Use bounded channel for backpressure (32 chunks = ~2MB buffer)
    let (response_sender, mut response_receiver) = tokio::sync::mpsc::channel::<Vec<u8>>(32);

    match call_t!(
        &state.router_ref,
        |reply| actors::router::RouterMsg::ExecuteStreamCommandRest {
            agent_id: agent.clone(),
            command: Command::RawDownload { path: path.clone() },
            reply,
            chunk_sender: response_sender,
        },
        30000
    ) {
        Ok(()) => {
            // Receive first chunk to check for errors
            let first_chunk = match response_receiver.recv().await {
                Some(chunk) => chunk,
                None => {
                    return (
                        StatusCode::INTERNAL_SERVER_ERROR,
                        Json(ErrorResponse { error: "No data received".to_string() }),
                    ).into_response();
                }
            };

            // Parse header to check for error
            let is_error = if first_chunk.len() >= redoor::streaming::HEADER_SIZE {
                first_chunk[21] == 1
            } else {
                return (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(ErrorResponse { error: "Invalid chunk received".to_string() }),
                ).into_response();
            };

            if is_error {
                return (
                    StatusCode::NOT_FOUND,
                    Json(ErrorResponse { error: format!("File not found: {}", path) }),
                ).into_response();
            }

            // Extract data from first chunk if present
            let first_data = if first_chunk.len() > redoor::streaming::HEADER_SIZE {
                first_chunk[redoor::streaming::HEADER_SIZE..].to_vec()
            } else {
                Vec::new()
            };

            // Create streaming body using async-stream
            use async_stream::try_stream;
            use futures_util::stream::Stream;
            use std::pin::Pin;

            let stream = try_stream! {
                // Yield first chunk data
                if !first_data.is_empty() {
                    yield bytes::Bytes::from(first_data);
                }

                // Stream remaining chunks
                while let Some(chunk) = response_receiver.recv().await {
                    if chunk.len() > redoor::streaming::HEADER_SIZE {
                        let data = &chunk[redoor::streaming::HEADER_SIZE..];
                        yield bytes::Bytes::copy_from_slice(data);
                    }
                }
            };

            Response::builder()
                .status(StatusCode::OK)
                .header("Content-Type", "application/octet-stream")
                .header("Content-Disposition", format!("attachment; filename=\"{}\"",
                    path.split('/').last().unwrap_or("file")))
                .body(Body::from_stream(stream))
                .unwrap()
                .into_response()
        }
        Err(e) => {
            let error_msg = format!("Failed to start stream: {:?}", e);
            (
                StatusCode::INTERNAL_SERVER_ERROR,
                Json(ErrorResponse { error: error_msg }),
            ).into_response()
        }
    }
}
```

## How Backpressure Works

### Server Side
1. **Slow HTTP client** → Axum stream buffer fills → HTTP read blocks
2. **Axum stream** → Waiting for `response_receiver.recv()`
3. **Bounded channel (32)** → When full, `response_sender.send()` blocks
4. **Router actor** → RouteStreamChunk handler blocks on send (line 345)
5. **Router** → Cannot process new chunks from WebSocket
6. **WebSocket** → TCP backpressure → Agent slows down sending

### Agent Side
1. **WebSocket send blocks** → `write.send(msg).await` waits (line 316)
2. **Channel consumer blocks** → `rx.recv().await` waits for send to complete
3. **Bounded channel (32)** → When full, `write.send()` on line 117 blocks
4. **Agent raw_download** → Pauses file reading until channel has space

**Memory usage**: Only ~2MB buffered (32 chunks × 64KB) on BOTH server and agent

## Summary

- **Server memory efficiency**: ~2MB buffer regardless of file size
- **Agent memory efficiency**: ~2MB buffer regardless of file size
- **End-to-end backpressure**: Bounded channels on both server and agent provide full flow control
- **Error handling**: First chunk check for proper 404 responses
- **No breaking changes**: Same API contract, just internal streaming
