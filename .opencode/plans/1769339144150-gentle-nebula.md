## Implementation Details

### Route Pattern
- Single route for both: `/agents/:agentId/browser/*`
- The `_splat` parameter can be either a directory path or file path

### Backend Data Flow
1. UI calls `/api/v1/agents/{agentId}/ls/{path}`
2. Backend `ls_agent_handler` sends `Command::Ls { path: Some(path) }` to agent
3. Agent's `ls()` command checks path type:
   - Directory: read directory, return `CommandResult::LsDirectory(LsDirectoryResult { files })`
   - File: get metadata, return `CommandResult::LsFile(LsFileResult { size, path, owner, group, uid, gid })`
4. Handler converts to appropriate response type:
   - `LsDirectory` â†’ `LsDirectoryResponse`
   - `LsFile` â†’ `LsFileResponse`

### Frontend Data Flow
1. User clicks file name in browser
2. Navigate to same browser route with agentId and file path
3. Loader calls `ls(path)` which returns union type
4. Component uses type guard to check response type
5. Renders either `FileList` (directory) or `FileDetailView` (file)

### Download URL Construction
Raw download URL format: `{baseUrl}/api/v1/agents/{agentId}/raw/{fullPath}`

Example: `http://localhost:8080/api/v1/agents/agent-1/raw/home/user/file.txt`
