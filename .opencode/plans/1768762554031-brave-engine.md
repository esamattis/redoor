# File Browser UI Implementation Plan

## Overview
Create a file browsing interface for agents that displays directory contents starting from the agent's CWD, supports navigation through nested directories, and maintains URL state matching the current path.

## Files to Create

### 1. `ui/src/routes/agents.$agentId.browser.tsx`
Main route file for the file browser.

#### Route Definition
```typescript
export const Route = createFileRoute("/agents/$agentId/browser/*")({
    loader: async ({ params, context }) => {
        const agents = await context.api.listAgents();
        const agent = agents.find((a) => a.id === params.agentId);
        if (!agent) throw new Error(`Agent not found: ${params.agentId}`);

        const details = await agent.getDetails();
        const relativePath = params._splat || "";
        const fullPath = relativePath
            ? `${details.cwd}/${relativePath}`
            : details.cwd;

        const lsResult = await agent.ls(fullPath);

        return {
            agentId: agent.id,
            agentName: agent.name,
            cwd: details.cwd,
            relativePath,
            fullPath,
            files: lsResult.files,
        };
    },
    component: FileBrowser,
    errorComponent: FileBrowserError,
});
```

#### Components

**FileBrowser** - Main component
- Displays header with breadcrumbs and navigation
- Shows file/directory table

**BrowserHeader** - Navigation header
- Breadcrumb navigation showing: Agent Name > path segments
- Up button (disabled when at CWD)
- Link back to agent details page

**Breadcrumbs** - Path breadcrumb navigation
- Each segment is a Link to navigate to that directory level
- Uses `getParentPath()` utility to build parent paths

**FileList** - Table of files and directories
- Header row: Type, Name, Size, Owner, Group
- ".." entry at top (if not at CWD)
- Directory entries first, sorted alphabetically
- File entries after directories, sorted alphabetically

**FileEntry** - Individual file/directory row
- Folder icon (blue, filled) for directories
- File icon (gray, outline) for files
- Name as clickable Link to navigate deeper (directories only)
- Formatted size (not shown for directories)
- Owner and group information

**FileBrowserError** - Error display
- Handle agent not found
- Handle path not found / not a directory
- Handle permission denied
- Generic error display with details

### Navigation Pattern

**Up Button:**
```typescript
<Link
    to="/agents/$agentId/browser/*"
    params={{ agentId, _splat: parentPath || "" }}
    className="flex items-center gap-2 px-4 py-2 bg-gray-100 rounded hover:bg-gray-200 disabled:opacity-50 disabled:cursor-not-allowed"
    disabled={isAtCwd}
>
    <ArrowUp className="h-4 w-4" />
    Up
</Link>
```

**Directory Navigation:**
```typescript
<Link
    to="/agents/$agentId/browser/*"
    params={{
        agentId,
        _splat: relativePath ? `${relativePath}/${entry.name}` : entry.name
    }}
    className="flex items-center gap-3 p-3 hover:bg-gray-50 rounded cursor-pointer text-blue-600"
>
    {entryIcon}
    <span className="font-medium">{entry.name}</span>
</Link>
```

## Error Handling

1. **Agent not found:** Display error message with link to agent list
2. **Path not found:** Display "Directory not found" error
3. **Not a directory:** Display error message
4. **Permission denied:** Display "Permission denied" error
5. **Generic error:** Show error message in centered error display
