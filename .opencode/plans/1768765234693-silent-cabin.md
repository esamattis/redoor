# Fix: Directory Listings Not Showing in File Browser

## Problem
The file browser at `/agents/$agentId/browser` is not showing directories. This is caused by a type mismatch between backend and frontend code.

## Root Cause
- **Backend (Rust)** returns `"directory"` for directory types (src/commands.rs:158)
- **Frontend (TypeScript)** checks for `"dir"` instead of `"directory"` (ui/src/routes/agents.$agentId.browser.tsx:35, 201)

## Implementation

### Changes Required

**File: `ui/src/routes/agents.$agentId.browser.tsx`**

1. **Line 35** - Update directory filter:
   - Change: `const directories = files.filter((f) => f.type === "dir");`
   - To: `const directories = files.filter((f) => f.type === "directory");`

2. **Line 201** - Update directory type check:
   - Change: `const isDirectory = entry.type === "dir" || isParent;`
   - To: `const isDirectory = entry.type === "directory" || isParent;`

## Verification

1. Start the development server: `./scripts/run-dev`
2. Navigate to: `http://localhost:4000/agents/agent1`
3. Click "Browse Files" button
4. Verify directories appear at the top of the file list (before regular files)
5. Click a directory to verify navigation works
6. Run tests: `./scripts/build-and-test`

## Expected Outcome
- Directories are displayed in file browser with folder icons
- Directories are sorted alphabetically and appear before regular files
- Clicking directories navigates into them (currently unimplemented but will show in list)
