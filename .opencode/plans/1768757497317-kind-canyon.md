# Plan: Improve API Client with Agent Class

## Objective
Refactor the API client to create an `Agent` class that encapsulates agent-specific operations, making the API more intuitive and object-oriented. Also create a DRY fetch wrapper to eliminate code duplication.

## Implementation Details

### 1. Create Fetch Helper Function
Create a private `fetch()` helper function that handles:
- URL construction (combining baseUrl with endpoint)
- Error handling (parsing ErrorResponse and throwing meaningful errors)
- Response JSON parsing

Function signature:
```typescript
private async fetch<T>(
    baseUrl: string,
    endpoint: string,
    options?: RequestInit
): Promise<T>
```

### 2. Create Agent Class
Add a new `Agent` class in `ui/src/api-client.ts` with:
- **Constructor**: Accepts `baseUrl: string` and `AgentInfoResponse`
- **Private helper**: Uses the `fetch()` helper to make requests
- **Properties**: `id` and `name` (readonly getters)
- **Methods**:
  - `getDetails(): Promise<AgentDetailsResponse>` - Get detailed agent information
  - `ls(path: string): Promise<LsResponse>` - List directory contents
  - `echo(message: string): Promise<EchoResponse>` - Echo a message

The Agent class will make HTTP requests directly using the fetch helper.

### 3. Refactor ApiClient Class
Modify `ui/src/api-client.ts`:
- Use the shared `fetch()` helper for all API calls
- Change `listAgents()` to return `Agent[]` instead of `AgentListResponse`
  - Create Agent instances with `this.baseUrl` and `AgentInfoResponse`
- Remove old public methods that take agent ID (`getAgentDetails()`, `ls()`, `echo()`)
- Update `waitForAgentNames()` to work with new `listAgents()` return type

### 4. Update UI Components
Migrate route loaders and components to work with new return type:

**File: `ui/src/routes/__root.tsx`**
- Remove destructuring: `const { agents } = Route.useLoaderData()` → `const agents = Route.useLoaderData()`
- Update sidebar navigation to use `agent.id` and `agent.name` directly

**File: `ui/src/routes/index.tsx`**
- Remove destructuring: `const { agents } = RootRoute.useLoaderData()` → `const agents = RootRoute.useLoaderData()`
- Update agent cards to use `agent.id` and `agent.name` directly

**File: `ui/src/routes/agents.$agentId.tsx`**
- Update loader to create Agent instance and use `getDetails()`:
  ```typescript
  loader: async ({ params, context }) => {
      const agents = await context.api.listAgents();
      const agent = agents.find(a => a.id === params.agentId);
      if (!agent) throw new Error(`Agent not found: ${params.agentId}`);
      return agent.getDetails();
  },
  ```

**File: `ui/tests/agents.test.ts`**
- Update to use Agent class for operations (getDetails, ls, echo)
- Example: `const agent = new Agent(apiClient.baseUrl, { id: agentId, name: 'test' })`

### 5. Import Addition
Add import for `AgentInfoResponse` type:
```typescript
import type { AgentInfoResponse } from "../../bindings/AgentInfoResponse";
```

## Benefits
- **Cleaner API**: No need to pass agent ID repeatedly
- **Better encapsulation**: Agent instances are self-contained and make requests directly
- **Extensibility**: Easy to add new agent methods in one place
- **Type safety**: Agent instances ensure operations are on the same agent
- **DRY code**: Fetch wrapper eliminates duplication in error handling and response parsing
- **Decoupled design**: Agent class doesn't depend on ApiClient
