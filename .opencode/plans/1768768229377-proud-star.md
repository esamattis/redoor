### Test Setup (beforeAll hook)

1. **Remove existing .test directory**: Use `fs.rm(TEST_DIR, { force: true, recursive: true })`
2. **Create .test directory**: Use `fs.mkdir(TEST_DIR)`
3. **Create directories**:
    - `subdir1/`, `subdir2/`, `subdir2/deep/`, `subdir3/`
4. **Create files**:
    - Root: `file1.txt`, `file2.txt`
    - `subdir1/`: `nested1.txt`, `nested2.txt`
    - `subdir2/deep/`: `nested3.txt`

### Test Scenarios

#### 1. File List at Agent Root

- Get first available agent using `ApiClient.listAgents()`
- Navigate to `/agents/{agentId}/browser/$`
- Verify `.test` directory is displayed in the file list
- Verify folder icon is shown

#### 2. Navigate to Subdirectory

- Click on `.test` directory
- Verify files `file1.txt` and `file2.txt` are displayed
- Verify subdirectories `subdir1`, `subdir2`, `subdir3` are displayed
- Verify correct count (2 files + 3 directories = 5 entries)

#### 3. Navigate Deeper

- Click on `subdir2`
- Verify `deep` directory is displayed
- Click on `deep`
- Verify `nested3.txt` file is displayed

#### 4. Breadcrumb Navigation

- From `subdir2/deep`, verify breadcrumbs show: `{agentName} / .test / subdir2 / deep`
- Click on `.test` breadcrumb
- Verify navigation back to `.test` root
- Click on `subdir1` breadcrumb
- Verify navigation to `subdir1` directory

#### 5. Up Button Navigation

- From `subdir2/deep`, click Up button
- Verify navigation to `subdir2`
- Click Up again
- Verify navigation to `.test`
- Verify Up button is disabled when at agent CWD

#### 6. Back to Agent Button

- From any directory, click "Back to Agent"
- Verify navigation to `/agents/{agentId}`

### Key Assertions

- File list displays correct count of files/directories
- Breadcrumbs show correct path hierarchy
- Directory names are clickable links with blue folder icon
- Files are not clickable and have gray file icon
- Size column shows "-" for directories, formatted size for files
- Up button disabled at agent CWD, enabled elsewhere
- Back to Agent button navigates correctly

## Implementation Notes

- Use `import { test, expect } from '@playwright/test'`
- Use `import fs from 'node:fs/promises'` and `import path from 'node:path'`
- Import `ApiClient` from `../src/api-client.ts`
- The web server (`../scripts/dev`) automatically starts and spawns local agents
- Tests will run at `http://localhost:4000/`
- Use `page.goto()` for navigation
- Use `page.click()` or `locator.click()` for interacting with elements
- Use `expect(locator).toBeVisible()`, `expect(locator).toHaveCount()`, etc. for assertions
- Follow the pattern: no sleep statements, use explicit waits for elements
