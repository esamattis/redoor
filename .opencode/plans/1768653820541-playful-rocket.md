## Implementation Details

### Process Management
- Use `child_process.spawn()` with `detached: true` to run processes
- Store PIDs for cleanup
- Use `SIGKILL` for immediate shutdown
- Make it a resuable class that can be configured with different server and agents args

### Startup Sequencing
1. Spawn server process
2. Poll server until it accepts connections (waitForPort helper)
3. Spawn agent process with ws://127.0.0.1:3000/ws and test-agent name
4. Wait 1 second for agent connection to establish

### Test Cases
Test only the ls API:
- Happy path: List directory contents on connected agent (200 OK)

### TypeScript Types
Import types from `/home/esamatti/code/redoor/bindings/`:
- `LsResponse` - { files: string[] }
- `ErrorResponse` - { error: string }
- Create simple api client (use classes, and fetch api)

### Configuration
- Test timeout: 30000ms
- Hook timeout: 10000ms
- Environment: node (required for fetch API and child_process)
