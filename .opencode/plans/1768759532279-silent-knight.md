# Plan: Move RouterMsg::GetAgentDetails to Command enum

## Objective

Simplify architecture by moving agent details query into the unified `Command` protocol, removing special-case handling from the router, and querying agents directly instead of using router cache.

## Current Architecture

```
REST API → RouterMsg::GetAgentDetails → Router Cache → Command::AgentInfo → Agent
```

- `RouterMsg::GetAgentDetails` queries router's cached registration data
- Router sends `Command::AgentInfo` to agent for runtime data
- Router combines cached (id, name, connected_at) + runtime data into response
- Uses separate `agent_details_pending_responses` map

## Target Architecture

```
REST API → RouterMsg::ExecuteCommandRest → Command::GetAgentDetails → Agent
```

- `Command::GetAgentDetails` added to command protocol
- Agent returns `CommandResult::GetAgentDetails(AgentDetailsResponse)` with runtime data
- Router fills in cached fields (id, name, connected_at) before returning
- Reuses existing `rest_pending_responses` map

```rust
async fn get_agent_details_handler(
    Path(agent): Path<String>,
    AxumState(state): AxumState<ServerState>,
) -> impl IntoResponse {
    match call_t!(
        &state.router_ref,
        |reply| actors::router::RouterMsg::ExecuteCommandRest {
            agent_id: agent.clone(),
            command: Command::GetAgentDetails,
            reply,
        },
        30000
    ) {
        Ok(result) => match result {
            CommandResult::GetAgentDetails(details) => {
                if details.name.is_empty() {
                    (
                        StatusCode::NOT_FOUND,
                        Json(ErrorResponse {
                            error: format!("Agent not found: {}", agent),
                        }),
                    )
                        .into_response()
                } else {
                    (StatusCode::OK, Json(details)).into_response()
                }
            }
            CommandResult::Error { message } => {
                let status = if message.contains("not found") {
                    StatusCode::NOT_FOUND
                } else {
                    StatusCode::INTERNAL_SERVER_ERROR
                };
                (status, Json(ErrorResponse { error: message })).into_response()
            }
            _ => (
                StatusCode::INTERNAL_SERVER_ERROR,
                Json(ErrorResponse {
                    error: "Unexpected result type".to_string(),
                }),
            )
                .into_response(),
        },
        Err(e) => {
            let error_msg = format!("Failed to get agent details: {:?}", e);
            (
                StatusCode::INTERNAL_SERVER_ERROR,
                Json(ErrorResponse { error: error_msg }),
            )
                .into_response()
        }
    }
}
```

## Benefits

1. **Simplified architecture** - Single unified command protocol for all agent operations
2. **Reduced complexity** - Removes special-case handling from router
3. **Consistency** - Agent details query follows same pattern as other operations (ls, cat, echo)
4. **Cleaner code** - Eliminates `agent_details_pending_responses` map and special handler

## Trade-offs

1. **Agent must be connected** - Cannot query cached info for disconnected agents
2. **Added network latency** - Requires WebSocket round-trip instead of reading from memory
3. **Agent complexity** - Requires agent to implement new command handler

## Rollback Plan

If issues arise:
1. Revert router.rs changes (restore `GetAgentDetails` variant and handler)
2. Revert main.rs changes (restore original handler)
3. Keep new `Command::GetAgentDetails` for potential future use
