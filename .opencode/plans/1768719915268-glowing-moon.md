# Redoor UI Implementation Plan

## Objective
Implement a real UI for redoor with a sidebar listing available agents and details panel for selected agents using only REST APIs.

## Current State
- Template TanStack Router application with minimal content
- REST API client already extracted to `redoor-ui/src/api-client.ts`
- All API types already generated in `bindings/` directory
- Tailwind CSS and lucide-react icons are installed

## Implementation Steps

### 1. Use Existing API Client

**Use:** `redoor-ui/src/api-client.ts` (already exists)
- Import `ApiClient` from `./api-client`
- Use `listAgents()` for sidebar
- Use `getAgentDetails(agent)` for agent details panel

### 2. Create Root Layout with Sidebar

**Modify:** `redoor-ui/src/routes/__root.tsx`
- Add loader to fetch all agents using `api.listAgents()`
- Create sidebar layout (280px width, full height, left side)
- Display agent list with clickable items
- Highlight currently selected agent based on route params
- Keep devtools in existing location

**Component structure:**
```tsx
export const Route = createRootRoute({
  loader: async () => api.listAgents(),
  component: RootLayout,
})
```

### 3. Implement Index Route

**Modify:** `redoor-ui/src/routes/index.tsx`
- Replace template content
- Access agent list from parent route using `useRouteContext()`
- Redirect to first available agent: `/agents/{agentId}`
- Show "No agents available" message if list is empty

### 4. Implement Agent Details Route

**Create:** `redoor-ui/src/routes/agents.$agentId.tsx`
- Route pattern: `/agents/:agentId`
- Loader: fetch agent details using `api.getAgentDetails(agentId)`
- Display agent details including:
  - Name and ID
  - Current working directory (cwd)
  - Process ID (pid)
  - Load averages (1min, 5min, 15min)
  - System uptime
  - OS and architecture
  - Hostname and username
  - Connected timestamp
- Use lucide-react icons for each detail section
- Handle loading states and errors
- Display "Agent not found" if request fails with 404

### 5. Error Handling

Three layers of error handling:
1. **API Client** - Throw descriptive errors for failed requests
2. **Loaders** - Let TanStack Router error boundaries handle thrown errors
3. **Components** - Check for error states, display user-friendly messages with retry buttons

### 6. Styling

Use Tailwind CSS for all styling:
- Sidebar: `w-72 border-r bg-gray-50`
- Main content: `flex-1 overflow-auto`
- Agent items: `px-4 py-3 hover:bg-gray-100 cursor-pointer`
- Selected agent: `bg-blue-50 border-l-4 border-blue-500`
- Agent details: Grid layout with cards for each section
- Detail cards: `bg-white rounded-lg border p-4`
- Detail items: `flex items-center gap-3 text-sm`
- Loading states: Tailwind spinner or skeleton

### 7. Testing

**Verify:**
- Run existing tests to ensure API client still works
- Start UI, verify agent list loads
- Click agent, verify agent details display
- Verify all detail fields show correctly
- Test empty agent list state
- Refresh page, verify URL-based navigation persists

After implementation:
1. Run `./scripts/build-and-test` to ensure tests pass
2. Start UI: `cd redoor-ui && pnpm dev`
3. Start Rust server (if not running)
4. Connect one or more agents
5. Verify agent list appears in sidebar
6. Click agent, verify agent details display in main area
7. Verify all detail fields show correctly (name, pid, cwd, load averages, etc.)
